# 📘 Day 9: LlamaIndex – Important Q&A

---

## ❓ 1. What is LlamaIndex?

LlamaIndex is a framework that simplifies building RAG systems by handling document processing, indexing, retrieval, and integration with LLMs.

---

## ❓ 2. Why do we use LlamaIndex?

We use LlamaIndex because it reduces boilerplate code, simplifies RAG development, and helps build scalable retrieval systems.

---

## ❓ 3. What is VectorStoreIndex?

VectorStoreIndex is an index structure that stores embeddings and enables vector-based retrieval using similarity search.

---

## ❓ 4. What are Nodes in LlamaIndex?

Nodes are the basic units in LlamaIndex representing chunks of text along with metadata and embeddings.

---

## ❓ 5. What is the role of a Retriever?

Retriever finds relevant chunks using embeddings and vector similarity search.

---

## ❓ 6. What is the role of Query Engine?

The Query Engine combines retrieval and LLM-based response generation to produce the final answer.

---

## ❓ 7. What is Metadata?

Metadata is additional information stored with nodes, such as:

* source file
* page number
* author

---

## ❓ 8. Why is LlamaIndex powerful?

LlamaIndex simplifies document processing and retrieval pipelines, making RAG systems easier to build and scale.

---

## 🎯 Summary

* LlamaIndex simplifies RAG development
* VectorStoreIndex enables semantic retrieval
* Nodes store text, metadata, and embeddings
* Query Engine handles retrieval + generation
