# 📘 Day 15: Evaluation Basics in RAG

---

## 🧠 What is Evaluation in RAG?

Evaluation means measuring how good a RAG system is.

👉 It checks:

* retrieval quality
* answer quality
* hallucination
* relevance
* faithfulness

---

## 🎯 Why is Evaluation Important?

A RAG system may:

* retrieve wrong chunks
* generate hallucinated answers
* provide irrelevant responses

👉 Evaluation helps identify these problems.

---

## 🔥 Two Main Parts of Evaluation

| Evaluation Type       | Checks                         |
| --------------------- | ------------------------------ |
| Retrieval Evaluation  | Are retrieved chunks relevant? |
| Generation Evaluation | Is the generated answer good?  |

---

# 🔹 1. Retrieval Evaluation

Retrieval evaluation checks whether the system retrieved relevant chunks for the user query.

---

## 🧠 Example

### Query

"What is deep learning?"

### Retrieved Chunk

"Deep learning uses neural networks."

✔ Good Retrieval

---

### Wrong Chunk

"Football is a sport."

❌ Poor Retrieval

---

## 🔥 Retrieval Metrics

---

### 🔹 Precision

Precision measures:
👉 Out of retrieved chunks, how many were actually relevant?

Formula:

Precision = \frac{Relevant\ Retrieved}{Total\ Retrieved}

---

### 🔹 Recall

Recall measures:
👉 Out of all relevant chunks available, how many were retrieved?

Formula:

Recall = \frac{Relevant\ Retrieved}{Total\ Relevant}

---

## 🧠 Precision vs Recall

| High Precision   | High Recall       |
| ---------------- | ----------------- |
| Less noise       | More coverage     |
| Strict retrieval | Broader retrieval |

---

# 🔹 2. Generation Evaluation

Generation evaluation checks the quality of the final answer generated by the LLM.

---

## Checks include:

* correctness
* relevance
* faithfulness
* groundedness
* hallucination

---

## 🔥 Hallucination

Hallucination occurs when the LLM generates incorrect or fabricated information.

---

### Example

### Retrieved Context

"Refund available within 7 days."

### LLM Answer

"Refund available within 30 days."

❌ Hallucination

---

# 🔥 Faithfulness

Faithfulness means:
👉 Is the answer supported by the retrieved context?

---

### Faithful Answer

Context:
"AI uses neural networks."

Answer:
"AI uses neural networks."

✔ Faithful

---

### Unfaithful Answer

"AI invented quantum computing."

❌ Unfaithful

---

# 🔥 Relevance

Relevance checks whether the answer actually addresses the user query.

---

### Example

Query:
"What is chunking?"

Answer:
"Chunking splits large text into smaller pieces."

✔ Relevant

---

# 🔥 Groundedness

Groundedness means the answer is based on retrieved information rather than imagination.

---

# 🔥 Human Evaluation

Humans manually evaluate:

* accuracy
* usefulness
* clarity
* relevance

---

## ⚠️ Problems

Human evaluation is:

* slow
* expensive
* difficult to scale

---

# 🔥 Automated Evaluation

Automated evaluation uses:

* metrics
* rules
* another LLM

---

## LLM-as-a-Judge

One LLM evaluates another LLM’s answer for:

* hallucination
* faithfulness
* quality

---

# 🔥 Common RAG Failures

| Failure           | Cause          |
| ----------------- | -------------- |
| Irrelevant answer | Poor retrieval |
| Hallucination     | Weak grounding |
| Incomplete answer | Low recall     |
| Noisy answer      | Low precision  |

---

# 🎯 Most Important Insight

Most RAG failures happen because of retrieval problems rather than only weak LLMs.

---

# 🔥 End-to-End Evaluation Flow

Query
→ Retrieval
→ Context
→ Generation
→ Final Answer Evaluation

---

# 🎯 Key Takeaways

* Evaluation measures retrieval and generation quality
* Precision and recall evaluate retrieval
* Faithfulness checks context alignment
* Hallucination detection is important in RAG
* Better retrieval improves answer quality

---

## 💡 My Understanding

Evaluation in RAG helps determine whether the system retrieves relevant context and generates grounded, accurate, and faithful answers.
